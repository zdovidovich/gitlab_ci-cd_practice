# CI/CD практика (GitLab CI)

Небольшая практика по написанию пайплайна в .gitlab-ci.yml. Код для сайта писал Qwen-3.7-Plus, **мои заслуги - Dockerfile и .gitlab-ci.yml**. Пайплайн выглядит так:

1. Линт Dockerfile с помощью `hadolint`
2. Тестирование с помощью pytest
3. Сборка `Docker` образа
4. Сканирование `Docker` образа с помощью `trivy` на известные уязвимости
5. Деплой на dev сервер, если коммит был в ветке dev
6. Деплой на prod сервер, если коммит был в ветке main. Требует ручного подтверждения

Также используются `CI/CD Variables` для приватных ключей `SSH` и IP-адресами хостов (к сожалению с доменными именами и моим dns сервером так и не захотел адекватно работать)
